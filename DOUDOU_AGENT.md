## 这是什么

**DouDou** 是一个运行在 [OpenClaw](https://github.com/openclaw/openclaw) 上的 AI Agent。它绑定一个微信账号，孩子发微信就能跟它对话学习。

**它不是什么**——不是 App、不是小程序、不是网页。孩子不需要学任何新软件，打开微信就能用。

### 三个文件定义一切

整个系统的逻辑浓缩在三个文件里：

| 文件 | 干什么 |
|------|--------|
| `SOUL.md` | Agent 的性格和说话方式 —— 温暖、简短、永远积极 |
| `USER.md` | 孩子的学习档案 —— 年级、进度、偏好 |
| `SKILL.md` | 辅导规则 —— 怎么出题、怎么打分、怎么记错题 |

**核心哲学：你是设计者，Agent 是执行者。** 规则你定，Agent 照做。

## 一句话价值

> **帮家长做"复习管理"这件事。**
>
> 学校教新知识没问题，老师讲、课本练。
> 但学完有没有复习？节奏对不对？弱点有没有反复练？
> 靠家长盯，精力跟不上。靠 7 岁孩子自觉，做不到。
>
> Agent 就是那个替你盯复习的管家。它记住每个知识点的状态，
> 按规则安排练习节奏，错的自动进复习池，对的逐步拉长间隔。

## 核心设计原则

### ⭐ 积分不是为了奖励，是"被看见"

积分的核心不是物质激励，是**让努力被看见**。每做完一题，Agent 立刻告诉她"本题 +5 分，累计 XX 分"。这个即时反馈比什么都重要。

**关键规则：**
- ✅ 完成一道练习 +5
- ✅ 全对额外 +3
- ✅ 连续学习第 N 天 +N（上限 +7）
- ❌ **永不扣分**——做错了只是没有全对奖励

### 📝 每次只做 1 题

一次出 5 题，孩子看到一堆题目就烦了。单题模式心理负担小很多——做完一道，加个分，开心一下，想继续就来一道。有时孩子做完数学意犹未尽，会主动说"再来一个"。

### 😊 绝不批评

做错了就说"差一点点！再来一次 💪"——绝不说"错了"。
**扣分**和**批评**是两件绝对不能做的事，前者破坏安全感，后者破坏学习意愿。

### 🔄 自动切换科目

孩子说"学数学"→ 切换数学模式；说"学语文"→ 切换语文模式。
如果没说具体，Agent 就问："今天想学语文📝、数学🧮、英语🔤还是科学🔬呀？🌈"

### 📊 数据驱动复习

每次练习后自动记录：练了什么、对错、得分。错题自动进复习池，每天优先从复习池出题，连续答对 3 次后移除。

## 交互流程

### 豆豆怎么说

```
"学数学" / "做数学"     → 进入数学模式
"学语文" / "认字"       → 进入语文模式
"学英语" / "读英语"     → 进入英语模式
"学科学" / "科学"       → 进入科学模式 🔬
"再来一个"              → 当前科目加一题，+1⭐
"不学了" / "不想"       → 结束学习，出今日小结
直接发答案              → 当前题目继续
```

### 核心文件详解

#### SOUL.md —— Agent 人格设定

定义了 Agent 的"说话方式"和"行为准则"：

```markdown
- 对豆豆：简短有趣，每句话不超过25个字，多用emoji
- 对家长（Yue）：正常汇报，把话说清楚
- 永远积极：做错了说"就差一点点！再来一次💪"
- 每次只教一件事：出1题/认1个字
- 不扣分、不批评、不比较
- 孩子说"不想学"→ "好呀，那休息一下😊"
```

#### SKILL.md —— 核心辅导规则

这是**最核心**的文件，定义了四个学习模式的所有逻辑：

- **数学模式**：每次 1 题，20 以内加减法，用孩子喜欢的角色代入出题。应用题场景化（小树上有 15 只鸟飞走了 7 只）
- **语文模式**：认字（出字卡→读出来→组词→造句）、古诗学习（教新诗→解释意思→跟读）
- **英语模式**：单词认读、简单问答（What's this? It's a cat 🐱）、拼读
- **科学模式**：趣味问答、判断题、选择题（青蛙几条腿？太阳从哪边升起？）

每个模式都有详细的交互示例和计分规则。

#### USER.md —— 孩子学习档案

个性化配置：
```markdown
- 年龄：7 岁 | 年级：一年级下学期
- 语文：人教版，当前进度第 47 页
- 数学：人教版，当前进度第 43 页
- 英语：朗文版，当前 Chapter 3
- 学习偏好：喜欢被怎么夸、注意力时长
```

## 知识库

### 语文（人教版一年级下册）

基于当前教材进度整理了字库和句型：
- **第 1-2 单元**：春、风、冬、雪、花、飞、入、姓、什、么...
- **第 3-4 单元**：青、清、气、晴、情、请、生、字、左、右...
- **第 5-6 单元**：蜻、蜓、迷、藏、造、蚂、蚁、食、粮...
- **常用组词**：春天、春风、春雨 / 花朵、开花、花园
- **句式训练**：______和______去______。/ ______在______。

### 数学（人教版一年级下册）

当前进度第 43 页（100 以内数的认识 & 认识人民币）：
- **20 以内退位减法题库**：11-2 到 18-9 全部覆盖
- **破十法讲解**：15-9 = 15-5-4 = 10-4 = 6
- **分类与整理**：按颜色、形状分类
- **100 以内数的认识**：数的组成、比较大小、整十数加减
- **认识人民币**：元角分换算
- **应用题题库**：6 道场景应用题（鸟飞走、糖吃掉等）

### 英语（朗文版一年级）

当前 Chapter 3，覆盖多种可能的主题：
- Animals / My Family / Colors / Numbers 词汇表
- Sight Words：a, the, I, my, you, your, he, she...
- 常见问答句型：What's this? / How are you? / Do you like cats?

### 科学（教科版一年级下册）

七个单元题库：
- 我们周围的物体（轻重、大小、形状）
- 动物（特征、分类、生活习性）
- 植物（根茎叶花果、生长条件）
- 天气与季节
- 我们的身体与五感
- 水
- 力与运动

## 积分系统

| 行为 | 分数 |
|------|:----:|
| 完成一道练习 | +5⭐ |
| 一次答对（全对奖励） | +3⭐ |
| 连续学习第 N 天 | +N⭐（上限 +7） |
| 主动要求"再来一题" | +1⭐ |
| 复习做错的旧题答对 | +2⭐ |
| **永远不扣分** | **0** |

每做完一题立刻显示：本题 +5，累计 XX 分，连续学习第 X 天。

## 错题复习池

自动机制：
1. **做错的题** → 自动记入 `memory/review-pool.md`
2. **每天复习时** → Agent 优先出复习池里的题
3. **连续答对 3 次** → 从复习池移除

复习池分为：数学错题 / 语文错字 / 英语错词 三部分。

## 部署方式

### 前置条件

1. **一台服务器**（腾讯云轻量 / AWS Lightsail / 任何 Linux 服务器，几十块/月即可）
2. **一个微信号**（用于绑定 Bot，推荐用注册的小号）
3. **Node.js 18+**

### 步骤

```bash
# 1. 安装 OpenClaw
npm install -g openclaw
openclaw setup

# 2. 配置微信 Bot
openclaw configure
# 选择 openclaw-weixin 插件
# 扫码登录微信 Bot 账号

# 3. 创建工作区
mkdir workspace-doudou
cd workspace-doudou

# 4. 添加 Agent
openclaw agents add doudou \
  --workspace ./workspace-doudou \
  --model deepseek/deepseek-reasoner \
  --bind openclaw-weixin

# 5. 将本仓库内容复制到 workspace-doudou/ 下

# 6. 重启生效
openclaw gateway restart
```

### 绑定微信账号

OpenClaw 支持 openclaw-weixin 插件，扫码登录后即绑定一个微信 Bot 账号。孩子的微信加这个 Bot 为好友，发消息给 Bot，Bot 自动转发给 DouDou Agent。

绑定配置在 `openclaw.json` 中：

```json
{
  "channels": {
    "openclaw-weixin": {
      "accounts": {}
    }
  },
  "bindings": [
    {
      "type": "route",
      "agentId": "doudou",
      "match": {
        "channel": "openclaw-weixin"
      }
    }
  ]
}
```

### 日常维护

- **更新学习进度**：修改 `USER.md` 中的教材页码
- **知识点调整**：更新 `knowledge/` 下的各科知识库
- **观察学习情况**：查看 `memory/progress.md` 和 `memory/review-pool.md`
- **修改出题规则**：编辑 `SKILL.md`

## 最佳实践 & 避坑

### ✅ 应该做的

1. **先跑最小版本**：先只配出题、答题、打分，跑两天看看孩子反应再加规则
2. **语音输入**：一年级打字慢，用微信自带语音转文字，或者装豆包语音输入法
3. **拍照导入**：直接拍课本词汇表照片发给 Bot，比自己打字快
4. **任务都要文字可完成**：别设计"朗读并录语音"之类链路太长的任务

### ❌ 别踩的坑

1. **一次设计太多规则再上线** → 先跑起来再迭代
2. **高估孩子对操作链路的耐心** → 全部文字完成
3. **出太多题** → 每次 1 题，不要贪多
4. **忘了更新教材进度** → 每次学完新单元要更新 USER.md

### ⚙️ 个性化配置建议

拿到源码后建议修改的地方：

1. **USER.md** → 孩子的名字、年级、教材版本和当前页码
2. **SOUL.md** → 如果孩子有昵称偏好或喜欢的称呼方式
3. **SKILL.md 中的角色代入** → 用孩子喜欢的动画角色出题
4. **积分上限** → 如果孩子学习动力强，可以调整分数
5. **知识库** → 换成你家孩子用的教材版本的内容
