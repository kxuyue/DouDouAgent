> 你不需要会编程，不需要搞服务器，有个微信就行

## 一、起因：辅导作业这件事，家长真盯不住

我女儿豆豆今年一年级下学期。

每天最头疼的不是她学不会，而是**忘了复习**。

学校里老师教得挺好，语文课上了，数学题做了，英语也读了。但回到家呢？
- 今天学了什么词？记不记得？
- 上周做错的题，这周还会不会？
- 今天该练什么？练多少？

我以前的做法是：下班回来问一句"今天作业做完了吗？"，她说"做完了"，我就信了。
一周后老师发通知说听写没过——哦，原来上周学的词早就忘了。

我也试过每天盯着她复习，但坚持了三天就累了。不是不想管，是真的**精力跟不上**。

后来看到一篇文章，有人用 OpenClaw 给他女儿搭了一套学习系统。我心想：这不就是我要的吗？

## 二、OpenClaw 是什么？跟 ChatGPT 有什么区别？

很多人第一反应是：用 ChatGPT 或者豆包辅导孩子不行吗？

能用，但用过就知道问题在哪。

**ChatGPT 是你问它答的模式。** 孩子不问，它就不动。可对一个 7 岁的小孩来说，"主动提问"恰恰是最难的部分。你总不能指望一年级小朋友每天自己打开 ChatGPT 说"请给我出三道 20 以内加减法吧"。

而且 ChatGPT 的聊天记录也没法沉淀下来做分析——上周错了哪些题？这个知识点练了几次？一片模糊。

**OpenClaw 不一样。**

它不是一个聊天窗口，它是一个**跑在服务器上的 AI Agent**。你可以给它写规则、设流程、建题库，它按照你定好的方式主动运转。

**你是设计者，它是执行者。**

打个比方：
- ChatGPT ≈ 随叫随到的解题高手
- OpenClaw ≈ 你雇了一个全职学习管家，而且这个管家严格按你写的手册办事

### 最关键的文件：就 3 个

**① SOUL.md — Agent 的人格设定**
```markdown
你是豆豆的学习伙伴，温暖、耐心、有趣。
- 说话简短，每句不超过 25 个字，多用 emoji
- 做错了说"差一点点！再来一次💪"，绝不批评
- 每次只出 1 题，做完问"还要再来吗？"
- 永不扣分
- 孩子说"不想学了"→"好呀，休息一下吧😊"
```

**② USER.md — 豆豆的学习档案**
```markdown
- 年龄：7 岁 | 年级：一年级下学期
- 语文：人教版，学到第 47 页
- 数学：人教版，学到第 43 页
- 英语：朗文版，Chapter 3
- 识字量约___ | 20以内加减法___
```

**③ SKILL.md — 核心辅导规则**

这是最核心的文件，定义了：
- **数学模式**：每次 1 题，20 以内加减法，用喜欢的角色代入出题
- **语文模式**：认字、组词、古诗学习
- **英语模式**：读单词、简单问答（What's this? It's a cat 🐱）
- **积分系统**：完成 +5⭐ / 全对额外 +3 / 连续学习第N天 +N / 永不扣分
- **错题自动进复习池**，答对 3 次后移除

## 四、几个关键设计原则

### ⭐ 积分不是奖励，是"被看见"

我一开始犹豫要不要做积分——怕搞成"做题换奖品"的交易。

后来想明白了：积分的核心不是物质激励，是**让努力被看见**。

每做完一题，Bot 立刻告诉她"本题 +5 分，累计 XX 分"。这个即时反馈比什么都重要。**孩子不怕辛苦，怕的是辛苦了没人知道。**

关键规则：
- ✅ 完成一道练习 +5
- ✅ 全对额外 +3
- ✅ 连续学习第 N 天 +N（上限 +7）
- ❌ **永不扣分**——做错了只是没有全对奖励，这个区别很微妙但很重要

### 📝 每次只做 1 题

之前试过一次出 5 题，豆豆看到一堆题目就烦了。改成单题模式之后，心理负担小很多——做完一道，加个分，开心一下，想继续就来一道。

有时候她做完数学意犹未尽，会主动说"再来一个"，这在以前是从来不会的。

### 📊 数据沉淀：从"感觉在学"到"看得见的进步"

这是我觉得最有价值的地方。

每次练习后，Bot 自动记录：练了什么、对错、得分。每周打开看：
- 这周练了哪些知识点
- 哪些反复出错
- 有没有在进步

以前辅导我最焦虑的就是**"不确定"**——感觉她好像在学，但到底学了什么、掌握了多少、哪里薄弱，全靠猜。现在有数据说话，下周重点练什么一目了然。

## 五、怎么自己搭一个？

### 你需要的东西
1. **一台服务器**（我用的腾讯云轻量服务器，几十块一个月）
2. **一个微信 bot 账号**
3. **安装 OpenClaw**（开源免费，一行命令搞定）

### 步骤

**第一步：装 OpenClaw**
```bash
npm install -g openclaw
openclaw setup
```

**第二步：配置微信 bot**
```bash
openclaw configure
# 选择 openclaw-weixin 插件
# 扫码登录微信 bot 账号
```

**第三步：创建工作区**
```
mkdir workspace-xiaoming（改成你家孩子的名字）
cd workspace-xiaoming
```

创建三个文件：
- `SOUL.md` — 人格设定（抄上面的改改就行）
- `USER.md` — 孩子学习档案
- `skills/grade1-coach/SKILL.md` — 辅导规则

**第四步：添加 Agent**
```bash
openclaw agents add 孩子昵称 \
  --workspace ./workspace-xiaoming \
  --model deepseek/deepseek-reasoner \
  --bind openclaw-weixin
```

**第五步：重启生效**
```bash
openclaw gateway restart
```

搞定。孩子现在发微信给 bot，就能学习了。

## 六、几个踩过的坑

**1️⃣ 规则别一次写完**
我一开始想把所有规则都设计好再上线，结果越想越复杂。后来改成**先跑一个最简版本**——就是出题、答题、打分——跑了两天，看看孩子的实际反应再慢慢加规则。

**2️⃣ 打字慢怎么办？**
一年级小朋友打字很慢。解决方案：用**语音输入法**（微信自带语音转文字，或者装豆包语音输入法），识别很精准。

**3️⃣ 别高估孩子对文字的耐心**
一开始我设计了一些"朗读并录语音"的任务，发现链路太长（录音→上传→转写→评估），她操作几次就烦了。后来**全部改成文字可完成的**，简单才能持续。

**4️⃣ 拍照比打字快**
我经常直接拍课本上的词汇表发给 Bot，它会自动识别整理入库，比自己手动打字快多了。

## 七、这套系统的本质

回头看，这套系统核心就解决一个问题：**怎么帮助孩子高效地复习。**

学校教新知识的部分没问题，老师会讲、课本会练。但学完之后有没有复习？复习的节奏对不对？薄弱的地方有没有反复练到？

靠家长人肉盯，精力跟不上。
靠孩子自觉，7 岁做不到。

OpenClaw 解决的就是这个"复习管不过来"的问题。它帮你记住每个知识点的状态，按规则安排练习节奏，错的自动进复习池，对的逐步拉长间隔。

孩子每天打开微信，不用想今天练什么——系统已经根据她的掌握情况安排好了。做完一道就有反馈，做错了马上纠正。

**这才是高效学习真正的样子——不是刷更多的题，而是在对的时间练对的内容。**
